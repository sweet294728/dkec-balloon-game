# DKEC 氣球特攻隊：分數進度、最後十秒助攻與排行榜設計

日期：2026-07-17

## 目標

在不改變既有計分、血量、速度與射擊規則的前提下，加入三項功能：

1. 結算時顯示距離下一等級尚差幾分。
2. 最後 10 秒依玩家分數增加氣球數量並提高正確目標比例，讓玩家更有機會達到 A 或 S 級。
3. 結算後可選填暱稱，將最佳成績送至 Google Sheet，並顯示前 20 名排行榜。

## 不變的既有規則

- 遊戲時間維持 60 秒。
- D、K 只能射 E、C；E、C 只能射 D、K。
- 正確氣球 +1 分；錯誤氣球 -1 分並扣 1 顆心。
- 愛心氣球在未滿血時補 1 顆心；滿血時 +1 分。
- 最多 5 顆心，血量歸零或時間歸零即結束。
- 氣球速度仍依原本時段使用 1、1.15、1.3 倍；最後 10 秒不額外加速。
- 新氣球仍從圍牆後方出現，完全通過可射擊起始線後才可命中。
- 等級門檻維持 D（低於 10）、C（10）、B（25）、A（40）、S（55）。
- 現有 D／C／B／A／S 獎勵連結維持不變。

## 一、距離下一等級

### 純規則介面

在 `src/game/rules.js` 新增 `getScoreProgress(score)`：

- 先以既有 `getScoreGrade(score)` 找到目前等級。
- D 的下一級為 C（10 分）。
- C 的下一級為 B（25 分）。
- B 的下一級為 A（40 分）。
- A 的下一級為 S（55 分）。
- `pointsToNext = nextGrade.minimum - score`。
- S 級沒有下一級，回傳 `isTopGrade: true`、`nextGrade: null`、`pointsToNext: 0`。

負分照實計算。例如 -2 分為 D 級，距離 C 級尚差 12 分。

### 結算畫面

在最終分數下方加入醒目的進度文字：

- 非 S 級：`距離下一等級 C 還差 3 分`。
- S 級：`恭喜！已達最高等級 S`。

文字必須有可供輔助科技讀取的完整語意，不只以顏色表達。

## 二、最後 10 秒自適應助攻

### 啟動條件

- `remainingSeconds > 10`：完全沿用現有難度。
- `0 < remainingSeconds <= 10`：啟用助攻。
- 遊戲結束後不再生成氣球。

### 目標分數

- 分數低於 40：助攻目標為 A 級 40 分。
- 分數介於 40～54：助攻目標為 S 級 55 分。
- 分數已達 55：視為已完成目標，維持較均衡的氣球內容。

### 同時存在的氣球數量

新增純函式 `getBalloonSpawnPlan({ elapsedSeconds, remainingSeconds, score })`。

一般時段回傳現有設定：

- 0～19 秒：6 顆，速度 1 倍。
- 20～44 秒：9 顆，速度 1.15 倍。
- 45～60 秒：12 顆，速度 1.3 倍。

最後 10 秒且尚未達助攻目標時：

```text
gap = goalScore - score
targetCount = clamp(14 + ceil(gap / 4), 14, 20)
preferredTargetRatio = 0.8
speedMultiplier = 1.3
```

範例：

| 當下分數 | 目標 | 差距 | 同時氣球數 |
| ---: | ---: | ---: | ---: |
| 0 | 40 | 40 | 20 |
| 25 | 40 | 15 | 18 |
| 39 | 40 | 1 | 15 |
| 40 | 55 | 15 | 18 |
| 54 | 55 | 1 | 15 |

分數已達 55 時：

- `targetCount = 14`
- `preferredTargetRatio = 0.5`
- `speedMultiplier = 1.3`

### 目標氣球比例

- 助攻期間，每次生成普通氣球時，有 80% 機率從玩家的兩個正確目標字母中選取。
- 其餘 20% 從另外兩個錯誤字母中選取，仍保留辨識與誤射風險。
- 已達 S 級時回到 50% 正確目標比例。
- 初始六顆氣球與前 50 秒的字母分布不變。
- 不直接加分、不自動命中、不改變飛鏢速度，也不保證完全沒有操作的玩家達標。

## 三、排行榜結算流程

### 使用者介面

在 `Results` 結算卡中，獎勵按鈕與重玩按鈕之間加入排行榜區塊：

1. 永遠可以查看前 20 名。
2. 暱稱為選填；不輸入不影響領獎、重玩或更換角色。
3. 暱稱輸入限制 1～12 個 Unicode 字元。
4. 按下「登錄排行榜」後進入送出中狀態，避免重複點擊。
5. 成功時顯示目前名次。
6. 若同暱稱已有更好的成績，顯示保留舊成績與舊名次。
7. 網路失敗時顯示可理解的錯誤訊息與「重新載入排行榜」按鈕，其他結算功能仍可使用。
8. 排行榜未設定 Apps Script 網址時，顯示管理者提示，不送出任何網路要求。

排行榜欄位：名次、暱稱、分數、等級、角色。手機版使用緊湊列式版面，不能產生水平捲動。

### 暱稱正規化與限制

- 顯示暱稱保留玩家輸入的大小寫。
- 比對暱稱時先做 Unicode NFKC 正規化、移除前後空白，再轉小寫。
- 正規化後相同即視為同一暱稱。
- 拒絕空白暱稱、超過 12 字、控制字元，以及以 `=`、`+`、`-`、`@` 開頭的內容，避免試算表公式注入。
- 前端先驗證以提供即時提示，Apps Script 後端必須再次驗證。

### 排名與最佳成績

整體排名依序比較：

1. 分數較高者優先。
2. 分數相同時，錯誤命中較少者優先。
3. 仍相同時，送出時間較早者優先。

同一正規化暱稱只保留一筆最佳紀錄：

- 新分數較高時取代舊紀錄。
- 同分但錯誤命中較少時取代舊紀錄。
- 其他情況保留舊紀錄。

## 四、Google Sheet 資料與 Apps Script

### 試算表

工作表名稱固定為 `Rankings`，第一列欄位為：

| 欄位 | 用途 |
| --- | --- |
| `recordId` | Apps Script 產生的唯一識別碼 |
| `submittedAt` | 伺服器寫入時間 |
| `nickname` | 顯示暱稱 |
| `normalizedNickname` | 去重用暱稱 |
| `score` | 最終分數 |
| `grade` | D／C／B／A／S |
| `characterId` | d／k／e／c |
| `correctHits` | 正確命中數 |
| `wrongHits` | 錯誤命中數 |

不收集姓名、Email、Google 帳號、IP 或其他個人資料。

### Apps Script 公開介面

`Code.gs` 提供：

- `setupLeaderboard()`：從綁定的 Google Sheet 取得試算表 ID，寫入 Script Properties，建立 `Rankings` 與標題列。
- `doGet(e)`：只處理 `action=list`，回傳前 20 名 JSONP。
- `doPost(e)`：只處理 `action=submit`，驗證成績並新增或更新最佳紀錄。

讀取採 JSONP，回傳內容僅包含公開排行榜資料；callback 名稱必須符合安全的 JavaScript 識別字格式。寫入採隱藏 HTML 表單 POST，Apps Script 回傳一個只負責 `postMessage` 的小型 HTML 頁面。前端以每次送出的 `requestId` 對應結果，收到成功訊息後重新讀取排行榜。

### 寫入安全與一致性

- 使用 `LockService.getScriptLock().waitLock(...)` 包住讀取、去重、寫入與排序，避免同時提交互相覆蓋。
- 伺服器只接受整數欄位與 d／k／e／c 角色。
- 分數、命中數與錯誤命中數設合理上限，攔截明顯異常資料。
- 等級一律由伺服器依分數重新計算，不信任前端傳入值。
- 回傳 HTML 內的資料先 JSON 編碼並避開 `</script>` 注入。
- 公開匿名端點無法完全防止玩家自行偽造請求；此版本只提供基本資料驗證與濫用防護，不宣稱具備完整防作弊能力。

## 五、前端模組與設定

預計新增：

- `src/leaderboard/config.js`：保存部署後的 `/exec` 網址；空字串代表尚未設定。
- `src/leaderboard/rules.js`：暱稱驗證、正規化與前端顯示用資料整理。
- `src/leaderboard/client.js`：JSONP 讀取、隱藏表單 POST、逾時與清理。
- `src/components/Leaderboard.jsx`：排行榜、暱稱輸入、送出狀態與錯誤狀態。
- `google-apps-script/Code.gs`：可直接貼入 Apps Script 的後端程式。
- `google-apps-script/部署說明.md`：從建立試算表到把 `/exec` 網址貼回遊戲的中文步驟。

預計修改：

- `src/game/rules.js`：分數進度與自適應生成規則。
- `src/game/engine.js`：依指定比例選取目標或錯誤字母。
- `src/game/GameCanvas.jsx`：把分數、剩餘時間與角色傳入生成規則。
- `src/components/Results.jsx`：顯示升級差距與排行榜。
- `src/styles.css`：結算進度、表單、排行榜與手機版樣式。
- `src/App.jsx`：把最終結果完整交給排行榜元件。
- `AGENTS.md`：移除與本次直接需求衝突的「不得加入排行榜」舊限制，其餘專案限制保留。

## 六、部署流程

1. 建立 Google Sheet。
2. 在試算表選擇「擴充功能 → Apps Script」。
3. 將專案提供的 `Code.gs` 完整貼入並儲存。
4. 在 Apps Script 編輯器手動執行一次 `setupLeaderboard()` 並完成授權。
5. 選擇「部署 → 新增部署作業 → 網頁應用程式」。
6. 執行身分選擇部署者本人；存取權選擇任何人。
7. 複製結尾為 `/exec` 的正式網址；`/dev` 僅供擁有程式編輯權的人測試，不可填入遊戲。
8. 將 `/exec` 網址貼入 `src/leaderboard/config.js`。
9. 執行完整測試、正式建置與單檔 HTML 匯出。
10. 將新版單檔 HTML 同步到既有網路資料夾。

Apps Script 後續修改時，從「部署 → 管理部署作業」建立新版本；保留原部署即可維持相同 `/exec` 網址。

## 七、錯誤處理

- 讀取逾時：顯示暫時無法連線並允許重試。
- 寫入逾時：解除送出中狀態，提醒玩家稍後重試。
- 無效暱稱：不送出請求，直接在欄位旁顯示原因。
- 後端驗證失敗：顯示後端回傳的安全中文訊息。
- Apps Script 未設定：不建立 JSONP 或表單，避免無意義錯誤。
- 排行榜故障：不得阻止領獎、再玩一次或更換角色。

## 八、測試與驗收

### 自動測試

- 每個等級與邊界分數的 `getScoreProgress`。
- 負分與 S 級最高等級訊息。
- 最後 10 秒啟動邊界（10 秒與 10 秒以上）。
- 0、25、39、40、54、55 分的氣球數量與目標比例。
- 助攻期間速度維持 1.3，不出現額外加速欄位。
- 80% 分支只產生正確目標，20% 分支只產生錯誤字母。
- 暱稱正規化、長度、控制字元與公式開頭驗證。
- 排行榜排序與同暱稱最佳紀錄規則。
- 未設定端點、讀取失敗與送出失敗狀態。
- 現有全部規則、碰撞、幾何、單檔匯出與正式版驗證測試不得退步。

### 瀏覽器驗收

- 桌面與手機結算畫面不溢出。
- S 級與非 S 級顯示正確進度文字。
- 暱稱可略過；錯誤內容能立即提示。
- 成功送出、保留舊最佳成績及連線失敗三種狀態皆清楚。
- 前 20 名在手機上可閱讀且沒有水平捲動。
- 領獎連結、再玩一次、更換角色仍可正常操作。
